私は Claude Code を **Claude Pro プラン（月額 $20）のサブスクリプション**で使っています。

「CLI ツール」「API キー」「コミット」といった言葉がよくわからなくても、普通の日本語で話しかけるだけで開発作業が進みます。この記事ではその感覚を伝えながら、使い始めるまでの手順を紹介します。

---

## Claude Code とは

ターミナル（黒い画面）上で Claude と会話しながら作業できる、Anthropic 公式のツールです。

- ファイルを実際に読んで編集してくれる
- エラーが出たら原因を調べて直してくれる
- 「GitHub に上げておいて」と言えば変更の記録から反映まで全部やってくれる

スマホで友人に「この設定どうすればいい？」と聞く感覚で、開発作業ができます。

---

## 前提：Pro プランについて

Claude Code を使うには **Claude Pro 以上のプラン**が必要です。

| プラン | 月額 | Claude Code |
|--------|------|-------------|
| Free | 無料 | 使えない |
| Pro | $20 | 使える |
| Max | $100〜 | より多く使える |

私は **Pro（$20/月）** で使っています。個人でブログ運営や日常的な開発をする分には十分です。使用量の上限に達したらその日は少し待つ必要がありますが、毎日ガッツリ使うのでなければ問題になりません。

## インストール

Node.js が必要です。まず確認します。

```bash
node --version
# v18 以上であればOK
```

入っていない場合は [nodejs.org](https://nodejs.org) からインストールしてください。

Claude Code のインストールはこの1行です。

```bash
npm install -g @anthropic-ai/claude-code
```

## ログインして起動する

インストール後、作業したいフォルダで `claude` を実行します。

```bash
claude
```

> **Windows の場合：PowerShell ではなく「コマンドプロンプト」で実行する**
>
> Windows では、PowerShell ではなく **コマンドプロンプト（Command Prompt）** で `claude` を実行する必要がありました。
>
> ターミナルパネルの右上にある **「+」ボタンの横の矢印（∨）** をクリックすると、起動するターミナルの種類を選べます。そこで「Command Prompt」を選ぶと切り替えられます。

### 初回：ターミナルでの初期設定

初めて `claude` を実行すると、ブラウザが開く前にターミナル内でいくつか設定を聞かれます。

**① テーマの選択**

```
? Select your preferred theme:
  1. Dark
  2. Light
  3. System
```

数字を入力して Enter を押します。あとから変更できるので、好みで選んで大丈夫です。

他にも確認項目が表示された場合は、基本的にデフォルト（Enter を押すだけ）で進めて問題ありません。

### 初回：ブラウザでのログイン

設定が終わると、ターミナルに以下のようなメッセージが表示されます。

```
No account found. Press Enter to log in with your browser.
```

Enter を押すと**ブラウザが自動で開き**、Claude.ai のログイン画面が表示されます。

1. Pro プランで使っているアカウント（メールアドレス＋パスワード、または Google/GitHub アカウント）でログイン
2. 「**Claude Code を承認しますか？**」という確認画面が出るので **「承認する（Allow）」** をクリック
3. 「ターミナルに戻れます（You can close this tab）」と表示されたらブラウザを閉じてOK

ターミナルに戻ると `>` のプロンプトが表示され、使える状態になっています。**API キーの設定は不要**です。

2回目以降は `claude` を実行するだけでログイン状態が維持されています。

## 実際の使い方：普通の言葉で話しかけるだけ

ここが一番伝えたいことです。

技術的な用語は一切知らなくて大丈夫です。やりたいことを日本語でそのまま書けば動きます。

### ファイルを触ってもらう

```
このフォルダの中身を見て、どんな構成か教えて
```

```
_config.yml というファイルのタイトルを「Kiwi Desk」に変えておいて
```

```
新しい記事のファイルを今日の日付で作って。タイトルは「NZで口座を開く方法」で
```

### エラーが出たとき

コマンドを実行してエラーが出たら、エラー文をそのままペーストして貼り付けます。

```
さっきこのエラーが出たんだけど直して

[エラーメッセージをペースト]
```

原因の調査から修正まで全部やってくれます。「なぜエラーが出たか」を説明してくれることも多いので、少しずつ理解が深まります。

### 変更を GitHub に反映する

「コミット」「プッシュ」といった用語を知らなくても大丈夫です。

```
ここまでの変更を保存して GitHub に反映しておいて
```

これだけで、変更内容の記録・説明文の作成・GitHub への反映まで全部やってくれます。

### ブログをローカルで確認したい

```
ブログをローカルで確認できるようにして
```

コマンドを調べる必要はありません。「ローカルで確認したい」という意図を伝えれば、必要なコマンドを実行してくれます。

## 知っておくと便利なこと

### `/compact` で頭を整理させる

長い作業をしていると Claude が前の内容を忘れてきます（コンテキスト上限）。そういうときは `/compact` と入力すると、会話を要約して続きをスムーズに進めてくれます。

### `!` で自分でコマンドを実行する

「このコマンドは自分で実行したい」という場面では `!` を先頭につけます。

```
! git status
```

実行結果がそのまま画面に表示されるので、「このエラーどういう意味？」と続けて聞くこともできます。

## 使ってみての感想

最初は「ターミナルで作業する」こと自体に抵抗がありました。ただ実際に使ってみると、**技術的な手順を調べる時間がほぼゼロになった**のが一番の変化です。

「こうしたい」を日本語で書く→ Claude が判断してファイルを操作する→結果を確認する、この繰り返しだけで作業が進みます。

このブログ自体も Jekyll の設定、記事ファイルの作成、GitHub へのデプロイまで Claude Code と一緒に作りました。具体的な作業の流れは別の記事で紹介します。
